Project Note

AWOO/KR을 만들며 배운 점

AWOO/KR은 이 포트폴리오의 전부가 아니라, 현재까지 가장 깊게 만들어본 대표 프로젝트입니다. 이 프로젝트를 만들며 서비스 기능 작성, 배포, 관리자 기능, 보안 관련 개념을 조금씩 경험해볼 수 있었습니다.

만들며 고민한 점

일반 회원 인증 없이 작성자가 본인의 글을 수정/삭제할 수 있게 하려면 어떻게 해야 할지 고민했고, 작성자 비밀번호를 hash로 저장해 확인하는 방식을 써봤습니다.

배포하며 배운 점

Vercel, Render, Supabase로 나누어 배포해보며 프론트엔드, 백엔드, DB, Storage가 어떻게 연결되는지 경험했습니다.

보안 관련 시도

CAPTCHA token 재사용 방지, 관리자 session cookie, CSRF token, 업로드 삭제 token 같은 개념을 프로젝트에 적용해봤습니다.

Project Note

스터디모임에서 연습한 기본기

규모를 크게 만들기보다 회원, 세션 인증, 게시글과 댓글, 검색과 페이지네이션처럼 웹 서비스에서 자주 만나는 흐름을 Spring MVC 구조로 나누어 구현하는 데 집중했습니다.

데이터 관계

회원, 모집글, 댓글과 답글, 좋아요, 북마크, 신고 관계를 JPA 엔티티와 MySQL 테이블로 구성했습니다.

인증과 보안

세션 로그인, BCrypt 비밀번호, 로그인 실패 제한, 이메일 인증번호 기반 비밀번호 재설정을 적용했습니다.

운영 기능

회원 제재, 콘텐츠 삭제, 신고 처리와 작업 기록을 확인할 수 있는 숨겨진 관리자 화면을 만들었습니다.
